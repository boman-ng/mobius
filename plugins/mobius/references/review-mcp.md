# Mobius Review MCP

Mobius Review is the recorded external review entry point for Mobius. Use it only for an explicitly
targeted Objective.

## Rules

- Every review is stateless.
- Every review receives a JSON Review Target generated by `review-target-create`.
- Each Review Target is one-shot input.
- CSV files and CSV rows are not MCP transport.
- Reviewers start from `ReviewTarget.refs`, `ReviewTarget.coverage`, and local read-only
  inspection.
- Prior review chat is not scope for a new review.
- `checkpoint_review` checks one Work Item.
- `exit_review` checks the full active Criterion matrix.
- Inline or loaded Review Targets must match the MCP tool's review mode.
- Missing, invalid, unchecked, degraded, or ambiguous reviewer output is not pass.
- Review Feedback is structured loop input, not Route failure budget.

## Tools

```text
mobius_review_health(deep?: boolean, include_commands?: boolean) -> mobius.review_health
mobius_review_registry() -> mobius.review_registry
mobius_review_build_subagent_prompt(review_target, review_mode) -> mobius.review_prompt
mobius_review_record_checkpoint_judgment(project_root, session_id, objective_slug, work_item_id, review_target|review_target_id, codex_subagent_result, level=1) -> mobius.review_result
mobius_review_record_exit_judgment(project_root, session_id, objective_slug, review_target|review_target_id, codex_subagent_result, level=2) -> mobius.review_result
```

The required host reviewer block is:

```text
MOBIUS_REVIEW_RESULT
REVIEWER: codex-subagent
REVIEW_MODE: checkpoint_review|exit_review
VERDICT: pass|fail|unknown|blocked
CHECKED_CRITERION_IDS: <json array>
UNCHECKED_CRITERION_IDS: <json array>
BLOCKING_FINDINGS: <json array>
REQUIRED_REVISIONS: <json array>
EVIDENCE_CHECKED: <json array>
FEEDBACK_ACTION: none|repair_route|add_evidence|select_alternate_route|retry_review|contract_change_required
NOTES: <short text>
END_MOBIUS_REVIEW_RESULT
```

The MCP persists valid output through the CLI, which owns ledger writes and Verdict derivation. It
does not edit CSV directly and does not convert reviewer prose into terminal state.

For `VERDICT: pass`, the result block must include all target Criteria in
`CHECKED_CRITERION_IDS`, an empty `UNCHECKED_CRITERION_IDS`, empty `BLOCKING_FINDINGS`, empty
`REQUIRED_REVISIONS`, and `FEEDBACK_ACTION: none`. Contradictory pass output is rejected before
any Review Judgment is persisted.

## Bundled Runtime

The plugin manifest points `mcpServers` at `./.mcp.json`, which defines the `mobius-review` stdio
server:

```json
{
  "mcpServers": {
    "mobius-review": {
      "cwd": ".",
      "command": "/bin/bash",
      "args": ["./scripts/mobius_review_mcp_server.sh"]
    }
  }
}
```

Set `MOBIUS_REVIEW_UV` to an absolute `uv` executable when `uv` is not on `PATH`. The launcher
uses `${PLUGIN_DATA}/uv-venv` when Codex provides `PLUGIN_DATA`; otherwise it falls back to a
user cache directory.
